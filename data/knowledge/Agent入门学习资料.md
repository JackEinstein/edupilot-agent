# Agent 入门学习资料

## 1. 什么是 Agent

Agent 通常指具备一定自主性、能够根据目标完成任务的智能系统。相比普通的大语言模型问答，Agent 不只是生成文本，而是可以围绕一个任务进行规划、调用工具、读取外部资料、执行步骤，并根据结果继续调整后续行为。

在大语言模型应用中，一个典型 Agent 通常包含以下能力：

1. 理解用户目标；
2. 拆解任务步骤；
3. 调用外部工具；
4. 读取和利用外部知识；
5. 保存必要的状态或记忆；
6. 输出最终结果。

因此，Agent 可以理解为“大语言模型 + 工具 + 状态 + 工作流”的组合。

## 2. Agent 和普通 Chatbot 的区别

普通 Chatbot 主要根据用户输入直接生成回答，流程通常比较简单：

用户问题 → 大模型 → 回答

Agent 的流程更加复杂：

用户目标 → 任务规划 → 工具调用 → 外部信息获取 → 中间结果判断 → 最终回答

例如，当用户问“帮我学习 LangGraph”时，普通 Chatbot 可能直接解释 LangGraph 的概念；而学习型 Agent 可以先制定学习计划，再检索相关资料，然后进行导师式讲解，最后生成复盘问题。

## 3. Agent 的核心组成

### 3.1 大语言模型

大语言模型是 Agent 的推理和语言生成核心。它负责理解用户输入、生成计划、分析资料、组织回答。

### 3.2 工具调用

工具是 Agent 获取外部能力的重要方式。工具可以是搜索引擎、数据库、计算器、代码执行器、文件读取器，也可以是本地知识库检索函数。

在 EduPilot Agent 项目中，Retriever 就是一种工具。它可以从 Chroma 向量数据库中检索与用户问题相关的学习资料。

### 3.3 工作流编排

复杂 Agent 往往需要多个步骤共同完成任务。工作流编排的作用是把这些步骤组织起来。

例如 EduPilot Agent 的流程可以设计为：

Retriever → Planner → Tutor → Reviewer

其中：

* Retriever 负责检索资料；
* Planner 负责制定学习计划；
* Tutor 负责导师式讲解；
* Reviewer 负责复盘和验收。

### 3.4 状态管理

Agent 在执行任务时，通常需要保存中间状态。例如用户问题、检索资料、学习计划、讲解结果、复盘结果等。

在 LangGraph 中，State 可以用来描述这些中间状态。每个节点读取 State 的一部分内容，并把新的结果写回 State。

### 3.5 记忆机制

Agent 的记忆通常可以分为短期记忆和长期记忆。

短期记忆用于保存当前会话中的上下文，例如用户刚才问了什么、系统刚才回答了什么。

长期记忆用于保存跨会话的信息，例如用户的学习偏好、知识薄弱点、长期学习目标等。

在 LangGraph 中，可以通过 checkpointer 保存短期状态。对于更正式的应用，可以使用数据库型 checkpointer 保存会话状态。

## 4. RAG 在 Agent 中的作用

RAG 是 Retrieval-Augmented Generation 的缩写，意思是检索增强生成。

普通大模型只能依靠自身参数中的知识回答问题，而 RAG 可以先从外部知识库中检索相关资料，再让模型基于资料回答。

RAG 的基本流程是：

1. 准备原始文档；
2. 对文档进行文本切分；
3. 使用 embedding 模型把文本块转成向量；
4. 将向量存入向量数据库；
5. 用户提问时，将问题也转成向量；
6. 在向量数据库中检索相似文本块；
7. 把检索结果作为上下文交给大模型；
8. 大模型基于上下文生成回答。

在 EduPilot Agent 项目中，本地 Markdown 和 txt 文件会被保存到 data/knowledge/ 目录，然后通过 Chroma 构建向量知识库。

## 5. LangGraph 在 Agent 中的作用

LangGraph 是一个用于构建多节点 Agent 工作流的框架。它适合把复杂任务拆成多个节点，并通过状态在节点之间传递信息。

在教育 Agent 场景中，LangGraph 可以让学习过程更加结构化。例如：

1. 先通过 Retriever 检索资料；
2. 再通过 Planner 生成学习计划；
3. 然后通过 Tutor 进行讲解；
4. 最后通过 Reviewer 检查学习效果。

相比单次 LLM 调用，LangGraph 的优势是流程更清晰、状态更可控、节点更容易调试。

## 6. EduPilot Agent 项目说明

EduPilot Agent 是一个面向学习场景的教育智能体 Demo。它的目标是帮助用户围绕某个学习主题完成学习闭环。

当前项目主要包含以下功能：

1. 用户输入学习问题；
2. 支持上传 Markdown 和 txt 学习资料；
3. 将学习资料保存到 data/knowledge/；
4. 重新构建 Chroma 向量知识库；
5. 根据用户问题检索相关学习资料；
6. 生成学习计划；
7. 进行导师式讲解；
8. 生成复盘和验收内容。

EduPilot Agent 的核心价值在于，它不是简单回答问题，而是模拟一个学习导师，帮助用户完成“资料检索—学习规划—知识讲解—复盘验收”的完整流程。

## 7. EduPilot Agent 的工作流程示例

假设用户输入：

我想学习 LangGraph 的多节点工作流，请帮我制定学习计划并讲解。

系统可以执行以下步骤：

1. Retriever 从知识库中检索 LangGraph 相关资料；
2. Planner 根据用户目标和资料生成学习计划；
3. Tutor 根据学习计划进行分层讲解；
4. Reviewer 生成复盘问题和掌握情况检查；
5. 前端展示检索资料、学习计划、导师讲解和复盘结果。

这种流程体现了 Agent 的核心思想：不是一次性回答，而是围绕目标组织多个步骤完成任务。

## 8. Agent 项目的应用价值

教育 Agent 可以应用于个性化学习、课程辅导、知识库问答、复习计划生成、错题分析和学习路径推荐等场景。

相比传统在线课程，Agent 可以根据用户输入动态调整讲解方式。相比普通搜索引擎，Agent 可以把检索结果组织成更适合学习的内容。相比普通 Chatbot，Agent 可以通过工作流拆解学习任务，使学习过程更加系统。

## 9. 未来优化方向

EduPilot Agent 后续可以继续扩展以下功能：

1. 增加 Quiz 节点，自动生成练习题；
2. 增加学习进度追踪；
3. 使用 LangGraph checkpointer 实现正式短期记忆；
4. 引入长期学习画像；
5. 增加多知识库管理；
6. 优化 RAG 检索效果；
7. 增加 rerank 模块；
8. 部署到云端，形成可在线访问的 Demo。

## 10. 小结

Agent 是大语言模型应用从“简单问答”走向“任务执行”的重要方向。一个完整的 Agent 通常需要模型、工具、状态、记忆和工作流共同配合。

EduPilot Agent 通过 LangGraph 实现多节点学习流程，通过 Chroma 实现本地知识库检索，通过 Streamlit 实现前端交互，形成了一个适合展示和扩展的教育智能体项目。
